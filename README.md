# Health Prediction App

A CRUD application for managing patient blood test records, with automatic 
health risk predictions generated by calling the Groq API (Meta's Llama 3.3 
large language model).

Built as a technical assessment task - backend in Flask, SQLite for storage, 
Bootstrap for the UI, and the Groq API for AI-powered health risk assessment.

## Features

- Create, Read, Update, Delete patient records
- Fields: Full Name, Date of Birth, Email, Glucose, Haemoglobin, Cholesterol, Remarks
- Server-side validation (valid email format, DOB can't be in the future, 
  blood values must be numeric and within clinically realistic ranges)
- Remarks are generated automatically by calling the Groq API — not typed by hand
- SQLite database for persistent storage
- API key managed via .env file, excluded from version control

## AI/ML API Integration

The app integrates with the Groq API, which provides access to Meta's Llama 3.3 
70B model. When a patient record is saved or updated, the application sends the 
blood test values to the API with a structured clinical prompt, and displays the 
returned health risk assessment in the Remarks field.

## Tech stack

- **Backend:** Python, Flask
- **Database:** SQLite via Flask-SQLAlchemy
- **Frontend:** HTML, Bootstrap 5 (Jinja2 templates)
- **AI/ML:** Groq API (Llama 3.3 70B model)

## How to run locally

1. Clone this repo:
git clone https://github.com/kirti2500/health-prediction-app.git
cd health-prediction-app

2. Create and activate a virtual environment:
python -m venv venv
venv\Scripts\activate   # Windows
source venv/bin/activate  # Mac/Linux

3. Install dependencies:
pip install -r requirements.txt

4. Create a `.env` file in the project root:
GROQ_API_KEY=your_groq_api_key_here
   Get a free API key at console.groq.com

5. Run the app:
python app.py

6. Open `http://127.0.0.1:5000` in your browser.
