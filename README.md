# Health Prediction App

A simple CRUD application that stores patient blood test records and uses
a trained machine learning model to generate a health risk prediction
("Remarks") for each patient.

Built as a technical assessment task - backend in Flask, SQLite for
storage, Bootstrap for the UI, and a logistic regression model
(scikit-learn) for the prediction part.

## Features

- Create, Read, Update, Delete patient records
- Fields: Full Name, Date of Birth, Email, Glucose, Haemoglobin, Cholesterol, Remarks
- Server-side validation (valid email format, DOB can't be in the future, blood values must be numeric and within clinically realistic ranges)
- Remarks are generated automatically using a trained ML model, not typed by hand
- SQLite database, no external services or API keys needed

## Why a custom ML model instead of a public API

I didn't have prior experience with external APIs, so I went with the
"custom ML model" option mentioned in the task brief instead. I trained
a logistic regression model on synthetic data based on common clinical
reference ranges for glucose, haemoglobin and cholesterol. This also
meant I didn't need to manage any API keys or worry about rate limits /
costs for this assessment.

See `train_model.py` for exactly how the training data and model were built.

## Tech stack

- **Backend:** Python, Flask
- **Database:** SQLite via Flask-SQLAlchemy
- **Frontend:** HTML, Bootstrap 5 (Jinja2 templates)
- **ML:** scikit-learn (Logistic Regression)

## How to run locally

1. Clone this repo and move into the folder:
git clone <repo-url>

cd health-prediction-app

2. Create and activate a virtual environment:
python -m venv venv

source venv/bin/activate   # Windows: venv\Scripts\activate

3. Install dependencies:
pip install -r requirements.txt

4. (Already done, but if you want to retrain the model yourself):
python train_model.py

5. Run the app:
python app.py

6. Open `http://127.0.0.1:5000` in your browser.
